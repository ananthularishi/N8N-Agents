# Agents Repository

This repository contains agent definition files for automation workflows, with a focus on school campaigns and education-related outreach.

## Repository Structure

- `School Email Campaign - Rishi (Fixed).json` — Agent configuration for school email campaign automation.
- `Telangana CBSE IIT Schools Agent - FIXED.json` — Agent configuration for outreach and engagement with Telangana CBSE/IIT schools.
- `N8N Agents/` — Additional automation workflows and related documentation for n8n.

## Purpose

These JSON files define agent settings and workflow parameters used by automation tools. They can be imported directly into the relevant platform or used as reference configurations for campaign setup.

## How to Use

1. Review the JSON file for the desired campaign or workflow.
2. Import the file into the automation platform that supports your agent definitions.
3. Verify the configuration matches your target audience and automation goals.
4. Update the file as needed and preserve version history through Git.

## Best Practices

- Keep agent definitions up to date with the latest workflow requirements.
- Use descriptive commit messages when modifying these files.
- Add additional documentation or example usage if a JSON file changes significantly.
